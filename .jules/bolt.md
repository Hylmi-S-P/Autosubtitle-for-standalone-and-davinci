## 2024-05-24 - Batching contentEditable input for subtitles
**Learning:** Frequent React state updates for a list of complex objects (`subtitles` array) from an `onInput` handler in a `contentEditable` div can lead to extreme lag and unresponsiveness, essentially blocking the main thread during quick typing. Updating files via Tauri adds extra overhead to the synchronous state update loop. Using a global debounce is prone to bugs like dropping edits when the user clicks another subtitle before the timer pops.
**Action:** Implemented a robust batching mechanism using `useRef` to track a Map of pending updates (`pendingUpdatesRef`), paired with a `flushUpdates` function. This queues edits across multiple indices immediately, and flushes them efficiently on a 500ms timeout *or* when clicking away/navigating/unmounting, ensuring no data loss and completely resolving the lag.
